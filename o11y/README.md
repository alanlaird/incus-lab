# o11y — Incus monitoring stack (Prometheus + Grafana)

Spins up two Alpine containers on the `cluster` remote and wires Prometheus to
scrape Incus `/1.0/metrics` over TLS, with Grafana pre-loaded with dashboard
19727 ("Incus — Overview").

---

## Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│  physnet (bridged)                                              │
│                                                                 │
│  ┌──────────────────┐          ┌────────────────────────────┐  │
│  │  prometheus      │          │  grafana                   │  │
│  │  172.16.11.190   │◄─────────│  172.16.11.110             │  │
│  │  :9090           │          │  :3000                     │  │
│  └────────┬─────────┘          └────────────────────────────┘  │
│           │ HTTPS (TLS client cert)                             │
│           ▼                                                     │
│  Incus cluster nodes :8443/1.0/metrics                         │
│  (172.16.11.141–144)                                           │
└─────────────────────────────────────────────────────────────────┘
```

| Container  | IP             | Port | Role                                      |
|------------|----------------|------|-------------------------------------------|
| prometheus | 172.16.11.190  | 9090 | Scrapes Incus metrics every 15 s          |
| grafana    | 172.16.11.110  | 3000 | Visualises metrics; dashboard 19727 built-in |

---

## How it works

1. **TLS metrics certificate** — Terraform generates a self-signed ECDSA cert
   (`metrics.crt` / `metrics.key`) and registers it with the Incus cluster as
   a read-only `--type=metrics` trust entry named `o11y-prometheus`.

2. **Prometheus** — installed from Alpine apk, configured to scrape all four
   cluster nodes at `https://<node>:8443/1.0/metrics` using the metrics cert.
   `insecure_skip_verify: true` is used because Incus nodes present self-signed
   certs.

3. **Grafana** — installed from Alpine apk, provisioned automatically with:
   - Datasource: the Prometheus container (source of truth is
     `grafana/datasource.yml`, generated by Terraform with the real IP).
   - Dashboard: Incus — Overview (ID 19727), downloaded from
     `grafana.com/api/dashboards/19727` and placed in
     `/var/lib/grafana/dashboards/incus.json`.

## Quickstart

```sh
make create        # terraform init + apply, then bootstrap
```

Then open Grafana:

```
http://172.16.11.110:3000
```

Login: **admin / admin** (Grafana will prompt you to change the password on
first login — do so before exposing this to any network).

The "Incus — Overview" dashboard is pre-provisioned. If you don't see it,
import dashboard ID `19727` manually (see below).

## Importing dashboard 19727 manually

If auto-provisioning did not work (e.g., grafana.com was unreachable during
bootstrap, or provisioning files landed in the wrong path):

1. Log in to Grafana at http://172.16.11.110:3000.
2. Go to **Dashboards → Import**.
3. Enter `19727` in the "Import via grafana.com" field and click **Load**.
4. Select the **Incus** datasource and click **Import**.

| Target      | Description                                                    |
|-------------|----------------------------------------------------------------|
| `help`      | List all targets with descriptions                             |
| `create`    | `apply` + `bootstrap` — full stack from scratch               |
| `apply`     | `terraform init` + `apply` (generates cert, creates containers)|
| `bootstrap` | Install prometheus + grafana, push configs, start services     |
| `list`      | Show running containers and access URLs                        |
| `destroy`   | `terraform destroy` + remove metrics cert from Incus trust     |

## Teardown

```sh
make destroy
```

This runs `terraform destroy -auto-approve` (removes the two containers,
generated local files, and the Terraform state entries) and then removes the
`o11y-prometheus` metrics certificate from the Incus trust store.

The `metrics.crt`, `metrics.key`, `prometheus.yml`, and
`grafana/datasource.yml` files are gitignored — they are regenerated each time
by `make create`.
