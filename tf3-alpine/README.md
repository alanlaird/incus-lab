# tf3 — Alpine VM Lab with Terraform + Ansible

Three Alpine Linux 3.21 virtual machines provisioned on a 4-node Incus cluster
via Terraform, then configured with Ansible.

---

## Prerequisites

| Tool      | Version tested | Notes |
|-----------|---------------|-------|
| Terraform | 1.14.6        | `brew install terraform` |
| Incus CLI | 6.22          | remote `cluster` configured at `172.16.11.141:8443` |
| Ansible   | 2.19.3        | virtualenv at `~/Sync/ivenv` |

The Incus `cluster` remote must already be authenticated (OIDC tokens at
`~/.config/incus/oidctokens`).  The `lxc/incus` Terraform provider reads the
same config directory.

```
│   └── playbook.yml       # Installs python3 and zsh

```

## Architecture

- **Image**: `images:alpine/3.21` (VM, not container)
- **Network**: `physnet` macvlan — VMs receive IPs on `172.16.11.0/24` via
  the upstream DHCP server, directly reachable from the management Mac.
- **Storage**: `local` ZFS pool, 4 GiB root disk per VM
- **Resources**: 1 vCPU, 512 MiB RAM per VM
- **Secure Boot**: disabled (`security.secureboot=false`) — required for
  unsigned Alpine images
- **SSH**: ED25519 key generated by Terraform (`tls_private_key`), injected
  via `incus exec` bootstrap (the Alpine 3.21 VM image does not include
  `cloud-init` or `openssh` by default)

## Quickstart

### 1. Initialise Terraform

```sh
cd tf3
terraform init
```

### 2. Create the VMs

```sh
terraform apply -auto-approve
```

Creates `alpine1`, `alpine2`, `alpine3` on the cluster and writes:
- `alpine_id_ed25519` — private key (0600)
- `alpine_id_ed25519.pub` — public key
- `ansible/inventory.ini` — Ansible inventory with actual IPs

If `ipv4_address` is null for a VM (DHCP race), run:

```sh
terraform refresh
terraform apply -auto-approve
```

### 3. Bootstrap SSH on the VMs

The Alpine 3.21 image is minimal and does not include `openssh-server`.
Bootstrap all three VMs at once:

```sh
PUB=$(cat alpine_id_ed25519.pub)
for vm in alpine1 alpine2 alpine3; do
  incus exec cluster:$vm -- sh -c "
    apk add --no-cache openssh-server &&
    ssh-keygen -A &&
    mkdir -p /root/.ssh &&
    echo '$PUB' > /root/.ssh/authorized_keys &&
    chmod 700 /root/.ssh && chmod 600 /root/.ssh/authorized_keys &&
    rc-update add sshd default &&
    rc-service sshd start
  "
done
```

### 4. Run the Ansible playbook

```sh
ansible-playbook -i ansible/inventory.ini ansible/playbook.yml
```

Installs **python3** and **zsh** on every VM.


## Teardown

```sh
terraform destroy -auto-approve
```

Destroys all three VMs and local key/inventory files.

## Known issues / notes

- **Alpine VM image is minimal** — no cloud-init, no sshd. The bootstrap
  step in §3 is required before Ansible can reach the VMs.
- **DHCP timing** — `ipv4_address` can be null immediately after VM creation
  on macvlan networks. A `terraform refresh` resolves this.
- **macvlan limitation** — VMs on `physnet` cannot communicate with the
  Incus cluster *host nodes* they run on (macvlan kernel restriction), but
  are fully reachable from the management workstation and from each other.
- **Terraform state holds the SSH private key** — treat `terraform.tfstate`
  as a secret; do not commit it.
